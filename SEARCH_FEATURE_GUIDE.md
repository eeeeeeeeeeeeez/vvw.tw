# 亨波 AI 代理：聯網搜尋功能完整指南

## 問題診斷：為什麼 AI 的年份資訊不準確？

### 根本原因
亨波 AI 使用的 Gemini 模型知識庫**截止於 2024 年**。因此：
- ✅ 2024 年及以前的資訊：準確
- ❌ 2025 年、2026 年的資訊：過時或錯誤
- ❌ 最新市場趨勢、新聞、價格：不可靠

### 解決方案：聯網搜尋功能
我們已實作了 **Google Search Function Calling**，使 AI 能夠：
1. **自動檢測**時間敏感的問題
2. **主動搜尋**最新資訊
3. **標註來源**並提供準確的年份數據

---

## 系統架構

```
用戶提問 (含年份關鍵詞)
    ↓
前端 (AI.tsx) → POST /api/ai/chat
    ↓
後端 (ai_v2.ts)
    ├─ 初始化 Gemini 模型 + Function Calling
    ├─ 系統提示詞：「當前年份是 2026 年」
    └─ 模型決策：
        ├─ 若需要最新資訊 → 調用 google_search 工具
        ├─ 執行 Google Custom Search API
        └─ 將結果整合至回覆
    ↓
前端串流接收結果 (包含搜尋過程)
    ↓
用戶看到最新、準確的資訊 ✅
```

## 環境配置要求

### 必需的環境變數
```bash
# .env.local

# Gemini API（必需）
GEMINI_API_KEY=your_actual_gemini_api_key

# Google Search API（用於聯網搜尋）
GOOGLE_SEARCH_API_KEY=your_google_search_api_key
SEARCH_ENGINE_ID=your_custom_search_engine_id
```

### 獲取 Google Search 金鑰步驟

#### 步驟 1：建立 Google Cloud 專案
1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立新專案（或選擇現有專案）
3. 啟用 **Custom Search API**

#### 步驟 2：建立 API 金鑰
1. 進入「認證」頁面
2. 建立「API 金鑰」類型的認證
3. 複製 API 金鑰

#### 步驟 3：建立 Programmable Search Engine
1. 前往 [Programmable Search Engine](https://programmablesearchengine.google.com/)
2. 建立新的搜尋引擎
3. **重要**：在設定中啟用「搜尋整個網路」選項
4. 複製 Search Engine ID (CX ID)

#### 步驟 4：設定環境變數
```bash
export GOOGLE_SEARCH_API_KEY="your_api_key"
export SEARCH_ENGINE_ID="your_cx_id"
```

## 常見問題 (FAQ)

### Q: 為什麼 AI 有時候還是給出 2024 年的資訊？
**A**: 有幾個可能原因：
1. 搜尋功能未正確配置（缺少 API Key）
2. 用戶的問題沒有明確包含時間關鍵詞
3. Google Search 沒有找到相關結果

**解決方案**：
- 確認 `GOOGLE_SEARCH_API_KEY` 與 `SEARCH_ENGINE_ID` 已正確設定
- 在提問時明確包含年份，例如「2026 年」
- 檢查後端日誌是否有搜尋錯誤

### Q: 搜尋功能會增加成本嗎？
**A**: 是的，但可控：
- Google Custom Search：每天 100 次免費查詢，超過需付費
- 我們已實作快取機制，相同查詢 1 小時內不重複搜尋
- 建議在生產環境監控 API 使用量

### Q: 能否限制搜尋範圍？
**A**: 可以。在 Programmable Search Engine 設定中，可以：
- 限制搜尋特定網域（如 .tw 台灣網站）
- 排除特定網站
- 設定語言偏好

### Q: 搜尋結果不準確怎麼辦？
**A**: AI 會自動重試：
- 若第一次搜尋結果不相關，AI 會更改搜尋詞重新搜尋
- 系統提示詞中的「觀察」階段會評估結果品質
